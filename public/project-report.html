📋 1. Executive Summary

GymCentre is a full-stack gym membership management platform built for Matoshree Fitness. It enables gym owners to manage members, track payments, handle membership plans, and digitize handwritten gym registers using AI-powered OCR. The application also provides a self-service member portal for members to check their membership status and manage UPI payment settings.

The platform is built with a modern tech stack — Next.js 16 with the App Router pattern, Supabase (PostgreSQL) as the backend, and integrates Google Gemini AI for handwriting recognition. It features a responsive design optimized for both desktop and mobile devices, with a production-hardened security setup including Row Level Security (RLS) and CI/CD pipelines.

🏗️ 4. System Architecture

Architecture Pattern

The app follows a Server-First Architecture using Next.js App Router. Data mutations are handled via Server Actions (located in app/actions/), while reads happen through direct Supabase client calls in client components or through server actions for optimized dashboard queries.

The system uses two Supabase client patterns:

  • lib/supabase/client.ts — Synchronous client for 'use client' components
  • lib/supabase/server.ts — Async client for server actions (uses cookies)

Authentication Architecture

Dual auth system — Admin Auth (email/password against an admins table) via AuthContext and AuthGuard, and Member Auth (phone-based login) via MemberAuthContext and MemberGuard/MemberPhoneGuard.

Row Level Security is enabled on all 8 public tables with policies restricting access to authenticated users only. Database indexes are applied on frequently queried columns (expiry_date, member_id, plan_id, payment_date).

🔒 10. Security Measures

  • Row Level Security (RLS) enabled on all 8 public tables — only authenticated users can access data
  • Auth Guards — AuthGuard for admin routes, MemberGuard & MemberPhoneGuard for member portal
  • Server-side validation — Zod schemas for input validation
  • UPI validation — Format validation before saving UPI IDs
  • Environment variables — Supabase keys via .env.local, not committed to git
  • Production build tested with CI/CD pipeline (GitHub Actions)
  • Database indexes on high-traffic query columns for performance

🖥️ 13. Setup & Running

Prerequisites

Node.js 18+, npm, a Supabase project with the schema applied.

Environment Variables (.env.local)

NEXT_PUBLIC_SUPABASE_URL=https://<project>.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
GEMINI_API_KEY=<your-gemini-api-key>

Commands

$ cd gym-centre
$ npm install — Install dependencies
$ npm run dev — Start dev server at http://localhost:3000
$ npm run build — Production build
$ npm run lint — ESLint check
